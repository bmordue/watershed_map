## 2026-02-24 - Accessible Interactive Map Layers
**Learning:** In geospatial visualizations using absolute-positioned elements (like DIVs or SVGs) to represent map layers, simply adding `onclick` handlers is insufficient for accessibility. These elements must have explicit `role="button"`, `tabindex="0"`, and `keydown` listeners (for Enter and Space) to be discoverable and operable by keyboard and screen reader users. Furthermore, `:focus-visible` styles on these absolute-positioned elements often require a high `z-index` and `outline-offset` to remain visible above overlapping map layers.
**Action:** Always implement a complete "accessibility triad" (semantic role, tab focus, and keyboard event mapping) for custom interactive geospatial elements. Ensure focus indicators use high-contrast colors and are prioritized in the stacking context.
